## 实现 VizSpec → HyperFrames 视频导出桥接管线

**目标**: 将 Get It. 的可视化产出（tags.json 中的 VizSpec）转换为 HyperFrames 可渲染的视频。

**创建的文件**:
- `lib/bridge-viz-to-hyperframes.ts` — 核心桥接模块
  - formula/graph/2d-text → 零成本直接映射为 HyperFramesScene[]
  - 3d/2d-anim → 通过 LLM 桥接 Agent（一次 Codex 调用）降级为 2D 场景描述
  - 定义 HyperFramesScene / HyperFramesElement 中间类型
- `lib/hyperframes-html-generator.ts` — HTML 生成器
  - 遵循 HyperFrames 规范：DM Sans + JetBrains Mono、暗色主题、GSAP timeline、HyperShader transitions
  - 支持 5 种 animation 模式 + 3 种 entrance 模式
  - 自动选择 shader transition（calm/medium）
- `app/api/export-video/[docId]/route.ts` — API 路由
  - mode=html: 返回 HTML 内联用于预览
  - mode=file: 写入 `<DATA_DIR>/docs/<docId>/hyperframes/` 供 `npx hyperframes render` 使用
- 修改 `components/RightPane/index.tsx` — 在"More"菜单中添加"Export Video"按钮

## 修复：Export Video 按钮无进度反馈

**问题**: 点击 Export Video 后 `setMoreOpen(false)` 立即关闭了菜单，进度文本全在菜单内部，用户看不到任何反馈。

**修复** (components/RightPane/index.tsx):
1. 移除 `exportVideo()` 开头的 `setMoreOpen(false)`，改为成功后 4s / 失败后 6s 自动关闭
2. subtitle 区域在 `exportingVideo` 时展示 `exportVideoPhase` 文本
3. 双重可见性：菜单内按钮文案 + header subtitle 同步显示进度

## 修复：export-video API 500 错误（spec 缺少 type 字段）

**根因**: tags.json 中 id=4-0 和 id=4-1 两个 tag 的 spec JSON 缺少 `type` 字段（LLM 生成时漏掉了 discriminator），导致 `bridgeSpecToScenes` switch 全部 fall through 返回 `undefined`，`bridgeAllTags` spread 时报 `TypeError: not iterable`。

**修复** (lib/bridge-viz-to-hyperframes.ts):
1. `bridgeSpecToScenes` 添加 `default` 分支返回 `[]`
2. `bridgeAllTags` 遍历前检查 `spec.type` 是否在 `VIZ_TYPES` 中，不在则跳过并 warn
3. 导入 `VIZ_TYPES` 常量用于运行时校验

## 添加标签选择导出功能

**需求**: 一个 PDF 有多个可视化，用户要挑其中一部分导出视频，而不是全量。

**修改**:
- `app/api/export-video/[docId]/route.ts`: POST body 新增 `tagIds?: string[]` 参数，传入时只导出选中 tag，不传仍导出全部
- `components/RightPane/index.tsx`:
  - 点击 "Export video" → 弹出标签选择器（按页分组，复选框）
  - "All ready" / "None" 快捷切换
  - ready 的 tag 可勾选，not ready 的灰显不可选
  - 底部 "Cancel" + "Export N tags" 操作按钮
  - 导出过程中菜单内显示 spinner + 实时进度文案

## 修复：export 成功后的反馈和输出路径不清晰

**问题**: Export 生成的是 HTML 文件（存本地磁盘），不是 MP4 浏览器下载。用户看不到输出在哪里，以为什么都没发生。

**修复**:
- 成功后不自动关闭菜单，显示绿色成功卡片
- 卡片含：场景数统计、完整 render 命令（可复制）、"Copy" 按钮、"Open folder in Finder" 按钮、"Dismiss" 按钮
- 失败后显示红色错误卡片，8 秒后自动关闭
- 区分三种状态：进度中(spinner) / 成功(绿色✓) / 失败(红色⚠️)
